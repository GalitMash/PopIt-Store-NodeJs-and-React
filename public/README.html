Specify all the different route your app supports

/: The products list
/cart: The products that the user added to his cart
/checkout: A page where you can see your items and checkout - pay (the cart is removed) or cancel checkout.
/customizepop: A page where you can send a form to purchase a custom Pop figure that will be customized according to the image you upload.
/storelocator: Displays our "physical stores" on a Google map. Please note that sometimes a "This page can't load Google Maps correctly." error appears and the map is grayed out, but it still works! just press "OK" in the error. This is just because I didn't want to give billing information to Google... :)
/admin : Displays the activity of all users, for the admin user only.
/login: User login page, with a "Remember me" option.
/signup: A page for registering a new user.

How did you make your store secured?

We validate all fields (including the image upload in Custom Pop) so no code can be injected/no harmful files can be uploaded. Also, there can be at most 50 files in our upload directory to prevent attackers from flooding us with uploads.
We authenticate users (their information can't be accessed or modified), and the admin screen can be accessed by the admin user only.

How did you make your store secured?

Forgery attacks- users cannot transmit unauthorized commands, since all input (in pages sign up and customize a pop) goes through input validation, to make sure that the input string is "clean" and nothing malicious will be written into the website files. also, you can't mofidy the admin user's password, because you can't override existing users.
DOS defense- there is no user input that can cause an exception and crash the site. since all input is validated, nothing bad can be injected. also when the user uploads a file in 'customize a pop', the file size is limited, and no more than 10 files can be uploaded.
